## 제4조 (음성 데이터의 처리)

본 조는 약관 제8조의 정책을 정보 처리 관점에서 보완합니다.

**(1) 학습 음성 — 즉시 폐기**

보이스 클론 생성을 위해 회원이 등록한 음성 샘플은 ElevenLabs API 로 음성 ID 가 생성된 직후 운영자 인프라에서 폐기됩니다 (저장 컬럼 및 Storage 버킷 모두 부재). 원본 학습 음성은 운영자가 보관하지 않으며, 재청취 기능도 제공하지 않습니다.

**(2) ElevenLabs 음성 ID — 외부 잔존**

ElevenLabs 가 보유하는 회원의 음성 ID 는 회원의 메시지 및 보이스 한마디 합성에 사용됩니다. 회원이 보이스 클론을 재녹음할 때 운영자는 이전 음성 ID 의 ElevenLabs API 삭제 요청을 발송하며, 회원 탈퇴 시에도 동일한 삭제 요청을 발송합니다. 삭제 요청의 최종 처리는 ElevenLabs 의 정책 및 처리 시한에 따릅니다.

회원은 보이스 클론을 단독 삭제할 수 없으며, 재녹음 (덮어쓰기) 또는 계정 탈퇴 경로로만 삭제할 수 있습니다. 이는 데이팅 서비스의 특성상 보이스 클론 임의 삭제로 인한 다른 회원의 매치 음성 재생 회귀 (영구 청취 불가) 를 방지하기 위함입니다.

**(3) 메시지 음성 — 30 일 자동 폐기 + 재합성**

채팅에서 합성된 메시지 음성 파일 (`voice-messages` 버킷) 은 **수신자가 청취한 시점으로부터 30 일이 경과** 하면 운영자의 정기 sweep 작업으로 자동 폐기됩니다. 메시지 텍스트 및 번역 결과는 매치 생명주기와 묶여 유지되며 음성 파일만 별도 폐기됩니다.

회원이 폐기된 음성을 다시 듣고자 할 때는 운영자의 재합성 라우트를 통해 ElevenLabs 로 새 음성 파일이 합성됩니다. 재합성된 음성은 회원의 현재 보이스 클론으로 합성되므로, 재녹음으로 음성 ID 가 변경된 경우 첫 발신 시점과 다른 음성이 합성될 수 있습니다 (의도된 정책).

미청취 메시지 음성은 30 일 경과 이후에도 자동 폐기되지 않습니다. 수신자가 청취 완료한 시점부터 30 일 카운트가 시작됩니다. 재합성된 음성도 동일한 정책이 적용되며, 재합성 시점을 기준으로 30 일 카운트가 다시 시작됩니다.

**(4) 보이스 한마디 음성 — 텍스트 변경/탈퇴 시 폐기**

보이스 한마디는 ko/ja/en 3 슬롯으로 합성되어 `voice-intro-audio` 버킷에 저장됩니다. 회원이 보이스 한마디 텍스트를 변경하면 이전 음성 파일은 자동 폐기되며 새 텍스트로 재합성됩니다.

## 제6조 (안전성 확보 조치)

운영자는 다음 조치를 통해 개인정보를 보호합니다.

1. 모든 API 통신은 HTTPS 로 암호화됩니다.
2. Supabase 의 Row Level Security 정책으로 회원 본인 외에 다른 회원의 정보 접근을 차단합니다.
3. 비밀번호는 Supabase Auth 에서 단방향 해시 (bcrypt 계열) 로 저장됩니다.
4. 콘텐츠 차단·정지 audit 테이블 (`moderation_blocks`, `freeze_events`) 은 service_role 권한으로만 접근 가능하며, 회원 본인을 포함한 어떤 회원에게도 노출되지 않습니다.
